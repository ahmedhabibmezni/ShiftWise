# FastAPI Framework
# Audit A14 — FastAPI >= 0.115 pulls Starlette >= 0.40, closing the
# multipart DoS CVE-2024-47874 in older Starlette releases.
# SV-012 — FastAPI 0.115.6 resolved Starlette ~0.41.x, still vulnerable to
# CVE-2025-54121 (MultiPartParser blocking/temp-file DoS, fixed in Starlette
# 0.47.2). Bumped to the version the test suite is actually validated against;
# it pulls a Starlette well past the fix. A Starlette floor is pinned so the
# fix can never regress on a transitive re-resolve.
fastapi==0.136.3
starlette>=0.47.2
# Audit G30 — uvicorn bumped 0.27.0 -> 0.47.0 (ASGI server only; no app
# API surface change). paramiko (3.4 -> 5.0) and the kubernetes client
# (28 -> 31) are now bumped (see their pins below): the SSH connectors use
# only paramiko's stable high-level API (SSHClient/connect/exec_command/
# open_sftp, password auth — no DSA keys, no disabled_algorithms) and the
# KubeVirt/migrator code uses only the stable CoreV1/BatchV1/StorageV1/
# CustomObjects client surface, both unaffected by the major jumps. The full
# test suite passes against the bumped versions.
uvicorn[standard]==0.47.0
# Audit A13 — python-multipart 0.0.18 fixed the ReDoS/DoS chain
# (CVE-2024-24762 / CVE-2024-53981). Bumped to 0.0.31 to clear a later
# path-traversal + DoS set (CVE-2026-24486 / -40347 / -42561 / -53538 /
# -53539 / -53540).
python-multipart==0.0.31

# SV-022 — passlib 1.7.4 is abandoned and pinned bcrypt to 4.0.1 (it reads the
# removed bcrypt.__about__ shim, breaking on bcrypt >= 4.1). We hash via the
# `bcrypt` library directly now (app/core/security.py), so passlib is dropped
# and bcrypt floats to a maintained release that can take future fixes.

# Audit G24 — cryptography is pinned explicitly (it is a transitive
# dependency of paramiko); an unpinned transitive makes the build

# 48.0.1 satisfies it; the Fernet vault uses only the stable high-level recipe

# SV-012 — urllib3 moved to the 2.x line now that the kubernetes client bump
# (28 -> 31, below) lifts the old <2.0.0 cap. 2.5.0 clears the full advisory
# set the 1.26.x line could not: CVE-2025-50181 / CVE-2025-50182 (redirect/
# PoolManager), CVE-2024-37891 (proxy-auth header leak on redirect), plus
# PYSEC-2026-141 / CVE-2025-66418 / CVE-2025-66471 / CVE-2026-21441 (fixed

# requires < 3, so 2.5.0 satisfies both.

# Kubernetes & OpenShift
# Bumped 28.1.0 -> 31.0.0 to MATCH the live cluster (OpenShift 4.18 = Kubernetes
# 1.31.5) — the client tracks the server minor, so 31.x is the exact-match,
# in-support pairing (vs jumping to 35/36, which would put the client 4-5 minors
# ahead of the server). Lifts the old urllib3 <2.0.0 cap (see urllib3 above),
# clearing that CVE set. The code uses only the stable CoreV1Api / BatchV1Api /
# StorageV1Api / CustomObjectsApi surface (KubeVirt CRDs go through
# CustomObjectsApi with an explicit group/version/plural — version-agnostic), so
# the bump is API-compatible. Validated by the full test suite.
kubernetes==31.0.0
pyyaml==6.0.1
